# DECISIONS — Bénévoles ACGB

Log des décisions techniques importantes.

| Date       | Décision                                                          | Contexte                                                                                                                                               | Alternatives considérées                                                                                                 |
| ---------- | ----------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------ |
| 2026-06-22 | Auth en **magic link** (Better Auth)                              | Public non-technique (membres de clubs), friction minimale, cohérent PWA                                                                               | Email + mot de passe (rejeté : trop de friction) ; nom+email sans compte (rejeté : notifs push fragiles)                 |
| 2026-06-22 | **Base Neon dédiée**, pas de couplage WordPress                   | MVP simple et découplé de acgb.ch                                                                                                                      | Branchement direct WordPress (rejeté : couplage fort) ; import CSV (reporté en option de lancement)                      |
| 2026-06-22 | SSO WordPress ACGB **reporté en V2**                              | Argument fort (« j'ai déjà un compte sur acgb.ch ») mais hors scope MVP                                                                                | Faire le SSO direct (rejeté : trop lourd pour un MVP)                                                                    |
| 2026-06-22 | Modèle **Poste → N Créneaux** (plage + capacité)                  | Besoin de flexibilité max (ex. buvette 7h-9h 2 pers., 9h-12h 1 pers.)                                                                                  | Créneau = 1 mission unique (rejeté : pas assez flexible)                                                                 |
| 2026-06-22 | **Push notifications dans le MVP**                                | Vrai intérêt du PWA : éviter les oublis d'inscription                                                                                                  | Reporter en V2 (rejeté : jugé important par le client)                                                                   |
| 2026-06-22 | Couleur de poste **auto-assignée**                                | Réduire la friction de création côté orga                                                                                                              | Sélecteur manuel (rejeté : friction inutile)                                                                             |
| 2026-06-22 | Email via **Resend**                                              | Choix explicite client                                                                                                                                 | Brevo (disponible mais non retenu)                                                                                       |
| 2026-06-22 | **PWA = service worker natif SvelteKit**                          | `@vite-pwa/sveltekit` incompat avec config-in-`vite.config.ts` (pas de `svelte.config.js`)                                                             | `@vite-pwa/sveltekit` (rejeté : fragile) ; `generateSW` (rejeté : pas besoin de Workbox au MVP)                          |
| 2026-06-22 | **PK uuid** (domaine) + `user.id` text                            | Liens publics non énumérables ; compat Better Auth (Epic 2)                                                                                            | serial (rejeté : énumérable)                                                                                             |
| 2026-06-23 | **`is_organizer` promu manuellement en DB**                       | MVP sans écran admin ; champ `input: false` côté client (non falsifiable)                                                                              | Allowlist email via env (reporté) ; premier compte = orga (rejeté : fragile)                                             |
| 2026-06-23 | **Prénom + nom sur le formulaire de login**                       | « email + nom au 1er usage » en un seul écran, zéro friction post-clic                                                                                 | Écran de complétion de profil après le 1er clic (rejeté : un écran de plus)                                              |
| 2026-06-23 | **Session 30 j** + magic link **15 min**                          | PWA reste connectée sur le téléphone ; lien assez long pour le mobile                                                                                  | Session courte (rejeté : reconnexions fréquentes)                                                                        |
| 2026-06-23 | **Login = action serveur, logout = client**                       | Validation Zod serveur ; signOut client gère le cookie via svelteKitHandler                                                                            | Tout côté client (rejeté : pas de validation serveur) ; logout action serveur (cookie plus délicat)                      |
| 2026-06-23 | **Heures de créneaux en UTC-naïf**                                | Heures « murales » stables : pas de dérive dev (CET) / prod Vercel (UTC) — suffixe `Z` stockage + `timeZone:'UTC'` affichage                           | Heure locale serveur (rejeté : dérive selon TZ) ; colonne `timestamptz` (rejeté : surdimensionné pour un MVP wall-clock) |
| 2026-06-23 | **Page de gestion orga unique inline**                            | PRD « rapide et simple » : empiler postes/créneaux sans navigation (`use:enhance`)                                                                     | Wizard multi-étapes (rejeté : plus de clics) ; une route par poste/créneau (rejeté : trop lourd)                         |
| 2026-06-23 | **Capacité = INSERT/UPDATE conditionnel SQL**                     | neon-http sans transactions ; check+write en une requête → pas de surréservation de la dernière place `available`                                      | Re-count puis insert (rejeté : course possible) ; transaction interactive (rejeté : indispo neon-http)                   |
| 2026-06-23 | **Noms des inscrits visibles sur /t/[token]**                     | Plus concret/social pour les bénévoles sous chaque créneau                                                                                             | Compteurs seuls (rejeté : moins concret)                                                                                 |
| 2026-06-23 | **Rappels push à 2 paliers : 24h + 2h**                           | Anti-oubli efficace (la veille + juste avant) ; idempotence via `reminder_24/2_sent_at` sur `signup`                                                   | Rappel unique (rejeté : moins efficace) ; délai unique configurable (reporté)                                            |
| 2026-06-23 | **Push self-hosted (web-push/VAPID) + cron polling**              | Volume minuscule (~120 push/tournoi) ; polling relit l'état → désinscription annule le rappel gratuitement                                             | Provider managé OneSignal (rejeté : overkill) ; scheduling pré-programmé QStash/Inngest (rejeté : annulations à gérer)   |
| 2026-06-23 | **Cron Vercel natif `*/15` (compte Pro)**                         | Pro autorise le sous-horaire → pas de déclencheur externe ; endpoint reste agnostique (`CRON_SECRET`)                                                  | Cron-job.org/GitHub Actions (gardé en repli) ; cron horaire (rejeté : fenêtre 2h trop lâche)                             |
| 2026-06-24 | **Impression planning via `window.print()` + `@page` A4 paysage** | Suivi orga imprimable sans dépendance ni génération PDF serveur ; 2 mises en page (par poste sans crop / matrice compacte)                             | Lib PDF côté serveur (rejeté : surdimensionné) ; capture image (rejeté : non sélectionnable/illisible)                   |
| 2026-06-24 | **Date picker = bits-ui + @internationalized/date**               | Remplacer l'`input date` natif (laid, variable navigateur) par un calendrier accessible on-brand ; contrat form `YYYY-MM-DD` préservé via hidden input | Calendrier custom maison (rejeté : accessibilité/clavier à refaire) ; rester en natif (rejeté : UX)                      |
| 2026-06-24 | **Durée tournoi = sélecteur 1j / 2j / Autre**                     | Cas courant (1-2 jours) réglé en un clic, fin dérivée du début ; « Autre » pour les rares plages longues                                               | Deux calendriers séparés (rejeté : friction sur le cas courant)                                                          |
| 2026-06-24 | **Suivi orga éditable (swap/move) — fin du lecture seule**        | Besoin d'ajuster les affectations ; bornage aux inscriptions existantes, gardé ownership, identifié par `(shiftId,userId)`, swap atomique 1 UPDATE     | Gestion complète (ajout/retrait — reporté) ; rester lecture seule (rejeté : besoin réel)                                |
| 2026-06-24 | **Pas de TanStack Table**                                         | La matrice est un cross-tab bespoke avec interaction clic→clic ; à notre échelle (≤~60 bénévoles) TanStack n'apporte rien et frictionne avec Svelte 5  | Adopter `@tanstack/svelte-table` (rejeté : zéro gain ici, dépendance + friction)                                        |
| 2026-06-24 | **Export `.xlsx` via `exceljs` (client)**                        | Vrai fichier Excel mis en forme (cellules colorées, en-têtes figées) ; import dynamique pour ne pas plomber le bundle                                  | SheetJS Community (rejeté : pas de styles de cellules) ; CSV multi-format (rejeté : pas de mise en forme)               |
| 2026-06-24 | **Téléphone obligatoire en validation (pas de `NOT NULL` DB)**   | Joignabilité ; éviter une migration `NOT NULL` risquée sur les comptes existants → garde-fou applicatif (bandeau + blocage inscription)                | `NOT NULL` + backfill (rejeté : risqué) ; tel optionnel (rejeté : besoin métier)                                        |
| 2026-06-24 | **Note libre par inscription, orga lecture seule**               | Le vrai planning encode tout dans les cellules (« dès 18h », « scoring ») ; visibilité publique = orga + propriétaire (via `organizerView`)            | Org-edit des notes d'autrui (reporté : conflit UX avec le clic-swap) ; notes visibles par tous (rejeté : bruit/privacy) |
| 2026-06-24 | **Matrice imprimée découpée par jour**                           | 43 créneaux = 43 colonnes → crop A4 paysage ; 1 bloc/jour (~20 col) tient ; par-poste rendu sécable (thead répété)                                     | Matrice unique (rejeté : crop) ; réduction de police seule (rejeté : illisible)                                         |
